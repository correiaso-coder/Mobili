O que é IA Generativa
Entenda o que faz a IA generativa ser diferente de qualquer software anterior — e por que isso muda radicalmente como automatizamos processos.
📖 Conceito Central

IA Generativa é uma categoria de Inteligência Artificial capaz de criar novos conteúdos — textos, imagens, código, áudio — a partir de padrões aprendidos em enormes volumes de dados. Diferente de sistemas baseados em regras fixas, ela raciocina por probabilidade e produz respostas contextuais e flexíveis.

Para negócios, isso significa automatizar tarefas que antes exigiam julgamento humano: redigir e-mails, interpretar reclamações, categorizar dados, gerar relatórios, responder clientes — tudo sem programar regras explícitas.

Diferença crucial: um sistema tradicional segue regras que você escreve. A IA Generativa aprende padrões dos dados e generaliza para situações novas — incluindo as que você não previu.
🧩 Os 4 Tipos de IA
🔮
IA Generativa
Cria conteúdo novo. ChatGPT, Claude, Gemini, DALL-E. Base dos agentes do projeto.
🔍
IA Discriminativa
Classifica e detecta padrões em dados existentes. Detecção de fraude, triagem de reclamações.
🎮
IA por Reforço
Aprende por tentativa e erro com recompensas. Base dos agentes autônomos avançados.
📊
IA Preditiva
Prevê resultados futuros com base em histórico. Churn, inadimplência, demanda.
🏢 Aplicações na Proteção Veicular
  • 01
    Atendimento automatizado: bot que responde dúvidas sobre cobertura, aciona sinistros e encaminha ao regulador — sem fila humana.
  • 02
    Triagem de Ouvidoria: IA classifica reclamações por urgência, sentimento e categoria antes de chegarem ao analista.
  • 03
    Geração de documentos: contratos, comunicados e relatórios gerados a partir de dados do CRM, sem digitação manual.
  • 04
    Análise de churn: identifica associados com risco de cancelamento e dispara régua de retenção personalizada.

LLMs e Modelos de Linguagem
Como os Grandes Modelos de Linguagem funcionam — o suficiente para usá-los bem e integrá-los em automações empresariais.
📖 O que é um LLM

LLM (Large Language Model) é um modelo treinado com bilhões de textos para prever qual token vem a seguir. Dessa capacidade emergem raciocínio, sumarização, tradução, geração de código e muito mais.

Quando você envia uma mensagem, o modelo processa seu prompt e gera tokens um a um — cada um influenciado pelos anteriores. A "inteligência" surge estatisticamente, não de regras.

Analogia: um LLM é como um estagiário que leu a internet inteira. Ele não "sabe" nada de forma definitiva, mas tem padrões sofisticados para gerar respostas úteis. Seu prompt é o briefing que determina a qualidade do resultado.
🔑 5 Conceitos Essenciais para Integração
  • Token
    Unidade básica de processamento. "automação" ≈ 3 tokens. Modelos têm limite de tokens por requisição (context window). Impacta custo e capacidade.
  • Temp.
    Temperatura controla criatividade. 0 = determinístico; 1 = criativo. Para automações empresariais use 0–0.3. Para geração criativa, 0.7–1.
  • Sys
    System Prompt é a instrução permanente que define o papel e comportamento do modelo. Equivale ao manual de atendimento do seu agente.
  • Ctx
    Context Window é a memória ativa de uma conversa. GPT-4o: 128k tokens; Claude: até 200k. Quanto maior, mais histórico o agente lembra.
  • Emb
    Embedding é a representação numérica de texto. Permite busca semântica — encontrar documentos similares mesmo com palavras diferentes.

O que é uma API
APIs são a cola de toda automação moderna. Entendê-las é o pré-requisito para conectar qualquer sistema.
📖 Definição Prática

API (Application Programming Interface) é um contrato de comunicação entre sistemas. Ela define o que você pode pedir, em qual formato, e o que receberá de volta.

Analogia do garçom: você não vai à cozinha do restaurante. Você faz o pedido ao garçom (API), que entrega o resultado. O garçom sabe como a cozinha funciona — você só precisa conhecer o cardápio (documentação).

Na prática da Mobili: quando um associado aciona o WhatsApp, o bot usa a API do WhatsApp para receber a mensagem, a API da OpenAI para gerar a resposta e a API do CRM para registrar o atendimento — tudo em segundos.

🔑 Anatomia de uma Chamada
  • URL
    Endpoint: o endereço do recurso. Ex: https://api.openai.com/v1/chat/completions
  • HTTP
    Método: a ação. GET (buscar) · POST (criar/enviar) · PUT (atualizar) · DELETE (apagar)
  • Auth
    Headers: metadados. O mais importante: Authorization: Bearer SEU_API_KEY — sua chave de acesso, nunca compartilhe.
  • Body
    Payload: os dados enviados em JSON. Usado em POST e PUT.
  • Resp
    Status Code: 200 OK · 201 Criado · 400 Requisição inválida · 401 Sem autenticação · 404 Não encontrado · 500 Erro no servidor

Postman na Prática
Postman é a bancada de testes. Antes de automatizar qualquer integração, você valida aqui.
🔧 Por que usar Postman

Antes de conectar uma API ao n8n, você precisa entender como ela funciona: quais campos enviar, o que ela retorna, como autenticar. O Postman permite fazer isso visualmente, sem escrever código.

Regra de ouro: sempre teste no Postman antes de automatizar. Um erro de JSON ou autenticação no Postman leva 2 minutos para identificar. No meio de uma automação, leva horas.

Introdução ao n8n
O n8n é o coração da Central Inteligente. É aqui que os fluxos ganham vida, conectando cada peça do ecossistema sem precisar programar.
📖 O que é o n8n

n8n é uma plataforma de automação de fluxos de trabalho open-source e visual. Você arrastra blocos (nodes), conecta com setas e define lógicas — sem código. Quando precisar de lógica avançada, suporta JavaScript nativo.

Por que n8n para a Mobili? Self-hosted = seus dados ficam no seu servidor. Para dados sensíveis de associados e sinistros, isso é crítico para conformidade e privacidade.
⚙️ n8n vs. Zapier vs. Make
Critérion8nZapierMake
CustoGrátis (self-hosted)Pago por tarefaPago por operação
Dados sensíveis✅ Self-hosted❌ Cloud apenas⚠️ Limitado
Lógica condicional✅ Avançada⚠️ Básica✅ Boa
Suporte a IA/Agentes✅ Nativo⚠️ Limitado⚠️ Parcial
🚀 3 Formas de Instalar
  • ☁️
    n8n Cloud (recomendado para início): acesse n8n.io, crie conta. Pronto em 2 minutos, sem configuração.
  • 🐳
    Docker local: docker run -it --rm -p 5678:5678 n8nio/n8n → acesse localhost:5678
  • 🖥️
    VPS (produção): Digital Ocean, Hetzner ou Railway. ~R$50/mês para servidor dedicado com dados protegidos.

Nodes, Triggers e Fluxos
A gramática do n8n. Entenda os blocos de construção para montar qualquer automação.
🧱 Os 4 Tipos de Nodes
⚡
Trigger Node
Inicia o fluxo. Webhook, schedule (cron), evento em app ou ativação manual.
⚙️
Action Node
Executa uma ação: envia e-mail, cria registro no CRM, chama uma API externa.
🔀
Logic Node
Controla o fluxo: IF (condição), Switch (múltiplos caminhos), Merge, Wait.
🔄
Transform Node
Transforma dados: Set (define campos), Function (JS customizado), JSON parse.
💡 Expressões n8n — Passando Dados
// Campo do node imediatamente anterior {{ $json.nome_associado }} // Output de node específico pelo nome {{ $node["CRM Request"].json.status }} // Operação condicional inline {{ $json.valor > 5000 ? "alto" : "normal" }} // Data atual formatada {{ $now.format("DD/MM/YYYY") }}

  • 🟢

  • 01

  • 02

    OpenAI recebe mensagem + contexto do associado e gera resposta personalizada.
  • 05
    HTTP Request envia a resposta de volta ao número no WhatsApp via API.
  • 06
    CRM registra a interação: data, canal, resumo, flag de escalada se necessário.

🔑 Endpoints Essenciais
EndpointUsoQuando usar

⚙️ Parâmetros Avançados do Chat
{ "model": "gpt-4o", // gpt-4o-mini para custo baixo "messages": [...], "temperature": 0.2, // 0=preciso, 1=criativo "max_tokens": 500, // limite de tokens na resposta "top_p": 0.9, // diversidade das opções (alternativa à temp) "frequency_penalty": 0.3, // reduz repetições "presence_penalty": 0.1, // incentiva tópicos novos "response_format": {"type": "json_object"} // força JSON válido }
response_format json_object é ouro para automações: garante que a resposta seja sempre JSON parseável, sem texto extra que quebre o fluxo.
💰 Controle de Custos
  • 💡
    Use gpt-4o-mini para classificação, triagem e tarefas simples. Custo ~20x menor que GPT-4o.
  • 💡
    Limite max_tokens — para triagem de reclamações, 100-200 tokens são suficientes. Para geração de relatórios, 800-1500.
  • 💡
    Cache do sistema: system prompts idênticos têm desconto de 50% no OpenAI. Padronize os prompts dos seus agentes.

Engenharia de Prompt
A habilidade que multiplica o valor de qualquer modelo. Um prompt bem construído transforma respostas mediocres em resultados profissionais.
📖 O que é Engenharia de Prompt

Engenharia de Prompt é a arte de estruturar instruções para extrair o melhor comportamento de um modelo de linguagem. A qualidade do output é diretamente proporcional à qualidade do input.

Princípio fundamental: trate o modelo como um funcionário brilhante que acaba de entrar na empresa. Ele é capaz, mas precisa de contexto claro, exemplos concretos e critérios explícitos de sucesso.
🏗️ Estrutura RECIPE para Prompts Profissionais
  • R
    Role (Papel): "Você é um especialista em atendimento ao cliente de seguros com 10 anos de experiência."
  • E
    Expectation (Objetivo): "Sua tarefa é classificar a reclamação a seguir em uma das categorias e avaliar a urgência."
  • C
    Context (Contexto): "A empresa é uma associação de proteção veicular. Os planos são: Basic, Standard, Premium."
  • I
    Instructions (Regras): "Nunca invente informações. Se não souber, responda 'informação não disponível'."
  • P
    Persona (Tom): "Tom: profissional, empático, sem jargões técnicos."
  • E
    Example (Exemplo): Forneça 1-2 exemplos do formato esperado de resposta.

Few-Shot e Chain-of-Thought
Duas técnicas avançadas que elevam drasticamente a precisão dos seus agentes em tarefas complexas.
🎯 Few-Shot: Aprender com Exemplos

Few-shot é fornecer exemplos de input/output no próprio prompt para que o modelo aprenda o padrão exato desejado. Mais eficaz que descrever o comportamento em palavras.

// Exemplo: classificação de sentimento para Ouvidoria { "role": "system", "content": "Classifique o sentimento da mensagem.\n\nEXEMPLOS:\nInput: 'Estou muito satisfeito com o atendimento'\nOutput: {\"sentimento\": \"positivo\", \"urgencia\": \"baixa\"}\n\nInput: 'Vou processar vocês, absurdo isso!'\nOutput: {\"sentimento\": \"negativo\", \"urgencia\": \"alta\"}\n\nClassifique a nova mensagem seguindo exatamente este formato." }
🔗 Chain-of-Thought: Raciocínio Passo a Passo

Chain-of-Thought (CoT) instrui o modelo a raciocinar explicitamente antes de responder. Para tarefas complexas, isso aumenta significativamente a precisão.

// Prompt com CoT para análise de sinistro "Analise o sinistro seguindo exatamente estes passos: PASSO 1 — Identifique o tipo de ocorrência PASSO 2 — Verifique se há documentação mencionada PASSO 3 — Avalie a urgência (P1=roubo/incêndio, P2=colisão, P3=demais) PASSO 4 — Determine o regulador responsável PASSO 5 — Retorne o JSON final com todos os campos Sinistro: {{mensagem_do_associado}}"
Quando usar CoT: decisões multi-critério, análise de elegibilidade de cobertura, triagem complexa de reclamações. Para tarefas simples, CoT é desnecessário e aumenta custo.

Function Calling
Function Calling permite que a IA decida quando chamar uma ferramenta externa — a base técnica dos agentes autônomos.
📖 O que é Function Calling

Com Function Calling, você descreve funções disponíveis (buscar no CRM, abrir sinistro, enviar e-mail) e o modelo decide quando e como chamar cada uma com base na conversa. O modelo não executa o código — ele retorna um JSON dizendo qual função chamar e com quais parâmetros.

Impacto real: é a diferença entre um chatbot que só responde e um agente que age. Com Function Calling, o agente pode consultar o CRM, abrir tickets, enviar notificações — tudo decidido pela IA com base no contexto.
🛠️ Estrutura de uma Function
{ "tools": [{ "type": "function", "function": { "name": "buscar_associado", "description": "Busca dados do associado no CRM pelo número de telefone", "parameters": { "type": "object", "properties": { "telefone": { "type": "string", "description": "Número com DDD, ex: 31999887766" } }, "required": ["telefone"] } } }] }

WhatsApp Business API
O canal de maior alcance no Brasil. Entenda a arquitetura oficial para integrações profissionais e conformes.
📱 Arquitetura da WhatsApp Business API

A API oficial da Meta permite automatizar mensagens no WhatsApp de forma escalável e dentro das políticas da plataforma. Diferente do app pessoal, ela exige aprovação da Meta e opera por provedores homologados (BSPs).

🏢
Meta (Cloud API)
Hospedada pela Meta. Sem servidor próprio. Mais simples de implementar. Recomendada para novos projetos.
🖥️
On-Premise API
Hospedada pelo próprio negócio. Mais controle, mais complexidade. Em descontinuação pela Meta.
🔗
BSP (Provedor)
Twilio, 360Dialog, Infobip, Zenvia. Intermediários que simplificam o acesso à API. Cobram por conversa.
⚡
Evolution API
Solução open-source não oficial. Ideal para testes e pilotos internos. Sem custo por mensagem.

Evolution API — Setup Completo
Suba sua própria instância da Evolution API para testes e desenvolvimento sem custo por mensagem.
🚀 Instalação com Docker
  • 01
    Certifique-se de ter Docker instalado. Em VPS Ubuntu: curl -fsSL https://get.docker.com | sh
  • 02
    Crie o arquivo docker-compose.yml com a configuração da Evolution API e Redis.
  • 03
    Execute: docker-compose up -d. Acesse: http://SEU_IP:8080
  • 04
    Crie uma instância via API: POST /instance/create com o nome da sua instância.
  • 05
    Busque o QR Code: GET /instance/qrcode/SEU_NOME e escaneie com o WhatsApp.
// Criar instância POST http://SEU_IP:8080/instance/create apikey: SUA_API_KEY { "instanceName": "mobili-atendimento", "token": "TOKEN_SECRETO", "qrcode": true }
🔗 Configurando Webhook para o n8n
// Configurar webhook para receber mensagens no n8n POST http://SEU_IP:8080/webhook/set/mobili-atendimento { "url": "https://seu-n8n.com/webhook/whatsapp", "webhook_by_events": true, "events": [ "MESSAGES_UPSERT", "CONNECTION_UPDATE" ] }
A partir deste ponto, toda mensagem recebida no WhatsApp conectado dispara automaticamente o webhook do n8n — iniciando o fluxo de atendimento inteligente.
